# Technical Storytelling Patterns

From analyzing effective technical blog posts.

## Narrative Frame

Open with personal context that transforms technical content into experience:
- "It's the holidays. Everyone's supposed to be taking a break. Instead, X messages me..."
- Hook via relatable interruption or moment
- Establish intimacy before diving into tech

## Sentence Rhythm

Vary length dramatically for pacing:

**Punchy fragments for emphasis:**
- "One tool call."
- "An hour. That's fast."
- "It worked."

**Then expand into complex explanations when needed.**

This alternation maintains engagement across dense material.

## Paragraph Structure

- Keep paragraphs concise (3-5 sentences)
- Prevent information overload
- Key concepts get their own section breaks
- Casual subheadings: "The Thing About X" not "Understanding X Architecture"

## Layered Technical Explanations

Don't define concepts academically. Instead:

1. Introduce abstractly
2. Ground in concrete scenario
3. Show measurable impact (98.7% reduction, 3x faster)
4. Code appears embedded in explanation, not isolated

**Bad:** "Tool calls are a mechanism by which..."
**Good:** "Here's the thing about tool calls - every time the model wants to do something, it costs tokens. We were burning through them just filtering documents."

## Parenthetical Personality

Add asides that show self-awareness:
- "(new)"
- "(exaggerating here but you get the drift)"
- "(I'll explain why in a sec)"

Acknowledges imprecision, builds trust.

## Progressive Disclosure

When addressing concerns:
1. Validate the concern first
2. Then immediately explain the solution
3. Don't let objections hang

- **Open:** Narrative hook, relatable context
- **Close:** Circle back to practical application
